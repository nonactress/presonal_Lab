# [문서 메타데이터]
- 출처: Marchionini (1995), "Information Seeking in Electronic Environments", Cambridge University Press
- 용도: 사용자가 디지털 환경에서 정보를 탐색하고 문제를 해결하는 인지적 과정을 분석하여 페르소나의 탐색 경로 및 막힘 현상 가설 설정
- 도메인: 전자 정보 시스템, 검색 엔진, 하이퍼텍스트 시스템

# CHUNK 1 — [정보 탐색의 본질: 문제 해결을 위한 동적 과정]
**검색 키워드 힌트:** 정보 탐색 과정, 지식의 차이, 문제 정의, 탐색 동기

정보 탐색은 사용자가 자신의 현재 지식 상태와 목표 지식 상태 사이의 '간극(Gap)'을 인식할 때 시작된다. 이는 단순한 데이터 검색이 아니라, 사용자가 자신의 멘탈 모델을 수정하거나 확장하려는 인지적 노력이다. 탐색 과정은 선형적이지 않으며, 새로운 정보를 발견할 때마다 목표가 수정되거나 탐색 전략이 바뀌는 매우 동적인 활동이다.

→ **페르소나 적용:** 페르소나가 검색창 앞에서 무엇을 입력해야 할지 고민하는 순간을 "자신의 멘탈 모델과 시스템 사이의 언어적 간극"으로 정의하고, 탐색 도중에 질문이 어떻게 구체화되는지 시나리오를 구성할 때 활용한다.

# CHUNK 2 — [전문가 vs 초보자의 탐색 스타일: 멘탈 모델의 차이]
**검색 키워드 힌트:** 검색 숙련도, 전문가 탐색, 초보자 탐색, 시스템 이해도

전문가는 시스템의 구조와 도메인 지식에 대한 명확한 멘탈 모델을 가지고 있어, 목표 지점에 도달하기 위한 효율적인 '작전(Operational)' 스타일을 구사한다. 반면 초보자는 시스템이 어떻게 작동하는지보다 '개념(Conceptual)'에 의존하며, 시행착오를 반복하는 탐색적 브라우징에 더 많은 에너지를 쓴다. 초보자는 시스템의 구조적 결함에 훨씬 더 민감하게 반응하며 쉽게 좌절한다.

→ **페르소나 적용:** "전문가 페르소나는 직접적인 검색어와 단축키를 선호하지만, 초보자 페르소나는 시스템이 제공하는 카테고리나 추천 리스트에 의존하며 길을 잃기 쉽다"는 가설을 세울 때 사용한다.

# CHUNK 3 — [인지적 저항의 법칙: 가장 쉬운 길을 찾는다]
**검색 키워드 힌트:** 최소 노력의 원칙, 인지적 비용, 탐색 경로 선택, 사용성 우선

사용자는 정보를 찾을 때 '최소 저항의 경로(Path of Least Cognitive Resistance)'를 선택하는 경향이 있다. 즉, 가장 정확한 정보를 제공하는 시스템보다 사용하기 가장 편리하고 인지적 노력이 적게 드는 시스템을 먼저 선택한다. 사용자는 정보의 질과 탐색에 드는 비용(시간, 정신적 에너지) 사이에서 끊임없이 타협한다.

→ **페르소나 적용:** 페르소나가 더 정확한 기능이 있음에도 불구하고 왜 자꾸 익숙하고 단순한 기능만 반복해서 쓰는지(기능 고착화)를 설명하는 가설로 활용한다.

# CHUNK 4 — [정보 탐색의 단계: 어디서 막히는가?]
**검색 키워드 힌트:** 탐색 단계, 쿼리 작성, 결과 검토, 정보 추출

정보 탐색은 [문제 인식 -> 시스템 선택 -> 쿼리 작성 -> 검색 실행 -> 결과 검토 -> 정보 추출 -> 반영]의 순환 구조를 가진다. 특히 초보 사용자는 '쿼리 작성(질문 던지기)' 단계와 '결과 검토(어느 것이 내게 필요한 것인가?)' 단계에서 가장 큰 인지적 부하를 느끼며, 여기서 기대와 다른 결과가 나오면 탐색을 포기할 확률이 높다.

→ **페르소나 적용:** 페르소나가 서비스 내에서 탐색을 멈추는 지점이 어디인지(예: 검색 결과가 너무 많아서, 혹은 원하는 용어가 없어서)를 단계별로 쪼개어 가설을 세울 때 활용한다.

# CHUNK 5 — [하이퍼텍스트와 브라우징: 자유와 혼란의 양면성]
**검색 키워드 힌트:** 하이퍼텍스트 탐색, 브라우징 행태, 비선형 탐색, 길 잃기 현상

웹과 같은 하이퍼텍스트 환경은 사용자에게 탐색의 자유를 주지만, 비선형적인 구조 때문에 '길 잃기(Disorientation)' 현상을 유발한다. 사용자는 자신이 전체 구조 중 어디에 있는지, 방금 어디서 왔는지 잊어버리게 되며 이는 탐색 효율을 급격히 떨어뜨린다. 잘 설계된 시스템은 사용자에게 현재 위치에 대한 시각적 단서를 지속적으로 제공해야 한다.

→ **페르소나 적용:** 페르소나가 여러 탭을 띄워놓거나 뒤로가기 버튼을 반복적으로 누르며 혼란스러워하는 상황(Context Lost)을 묘사하고, 이를 해결하기 위한 네비게이션 가이드 가설을 도출할 때 사용한다.
