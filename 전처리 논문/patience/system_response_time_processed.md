# [문서 메타데이터]
- 출처: Hoxmeier (2000), "System Response Time and User Satisfaction: An Experimental Study of Browser-based Applications", AMCIS 2000 Proceedings
- 용도: 시스템의 응답 속도가 사용자의 만족도와 서비스 이탈에 미치는 임계치를 파악하여 페르소나의 인내심 및 행동 패턴 가설 설정
- 도메인: 브라우저 기반 애플리케이션 및 웹 서비스

# CHUNK 1 — [응답 시간과 만족도의 관계: 속도가 느려지면 만족도는 떨어진다]
**검색 키워드 힌트:** 시스템 응답 시간, 사용자 만족도, 로딩 속도, 서비스 불만

시스템 응답 시간(System Response Time)은 사용자가 작업을 시작한 순간부터 결과가 화면에 나타날 때까지의 시간이다. 연구 결과, 응답 시간이 길어질수록 사용자의 주관적인 만족도는 선형적으로 감소한다. 특히 응답 시간이 3초를 넘어가면서부터 만족도의 하락이 뚜렷하게 관찰되며, 이는 서비스 재사용 의도에 직접적인 영향을 미친다.

→ **페르소나 적용:** 페르소나가 특정 버튼을 클릭하거나 페이지를 이동할 때, 3초 이상의 대기 시간이 발생하면 서비스에 대한 신뢰를 잃고 짜증을 느끼기 시작한다는 가설을 세울 때 활용한다.

# CHUNK 2 — [사용자 이탈의 임계치: 12초의 법칙]
**검색 키워드 힌트:** 사용자 이탈 임계치, 대기 시간 한계, 서비스 포기 시점, 12초

자율적인 선택권이 있는 사용자(Discretionary User)의 경우, 시스템 응답 시간이 약 12초에 도달하면 서비스를 더 이상 사용하지 않겠다고 결정하는 임계점에 도달한다. 12초는 사용자가 "이 시스템은 고장 났다"거나 "기다릴 가치가 없다"고 판단하여 대체 수단을 찾게 만드는 결정적인 시간이다.

→ **페르소나 적용:** 페르소나가 복잡한 계산이나 데이터 로딩을 기다릴 때, 12초가 넘어가면 탭을 닫거나 다른 서비스로 넘어가는 행동 패턴을 정의할 때 기준 수치로 사용한다.

# CHUNK 3 — [응답 시간과 지각된 용이성: 느린 것이 곧 어려운 것이다]
**검색 키워드 힌트:** 지각된 용이성, 인터페이스 반응성, 사용 편의성 오해

사용자는 시스템의 응답이 느릴수록 해당 시스템을 사용하기 더 어렵다고(Ease of Use가 낮다고) 인식한다. 시스템의 기능적 난이도가 동일하더라도, 단순히 응답 속도가 느린 것만으로도 사용자는 "시스템 조작이 복잡하다"거나 "배우기 힘들다"고 느끼게 된다. 만족도의 하락은 곧 사용성(용이성)의 하락으로 이어진다.

→ **페르소나 적용:** 페르소나가 인터페이스를 '불친절하다'고 느끼는 이유가 기능의 복잡성문제가 아니라, 느린 반응 속도 때문일 수 있다는 UX 가설을 도출할 때 사용한다.

# CHUNK 4 — [학습 효과와 응답 시간: 너무 빠르면 오히려 방해될 수 있다]
**검색 키워드 힌트:** 시스템 학습, 즉각적 응답 부작용, 인지적 처리 시간, 서두름 현상

흥미롭게도 시스템 학습(System Learning) 측면에서는 즉각적인 응답(0초 대기)이 반드시 최선은 아니다. 응답 시간이 약간 있는 경우 사용자는 앞서 수행한 작업을 정리하고 다음 단계를 준비할 시간을 갖게 되지만, 너무 빠른 응답은 사용자를 '서두르게' 만들어 실수를 유발할 수 있다. 적절한 지연은 오히려 사용자가 시스템 운영 방식을 이해하는 데 도움을 줄 수 있다.

→ **페르소나 적용:** 초보 페르소나가 튜토리얼을 진행하거나 새로운 기능을 익힐 때, 너무 빠른 화면 전환보다는 사용자가 정보를 처리할 수 있는 적절한 시각적 피드백과 여유를 제공해야 한다는 설계 원칙을 도출할 때 활용한다.

# CHUNK 5 — [숙련도에 따른 인내심 차이: 전문가는 더 잘 기다리는가?]
**검색 키워드 힌트:** 사용자 숙련도, 기술 경험, 응답 시간 인내심, 전문가 vs 초보자

연구에 따르면 인터넷 사용 경험이 많은 숙련된 사용자와 초보 사용자 사이에 응답 시간에 대한 인내심(Tolerance) 차이는 유의미하지 않았다. 즉, 인터넷에 익숙한 사람이라도 시스템이 느린 것에 대해 초보자와 마찬가지로 똑같이 짜증을 느끼며, 경험이 많다고 해서 느린 시스템을 더 잘 참아주지는 않는다.

→ **페르소나 적용:** "전문가용 툴이니까 사용자가 로딩을 잘 참아줄 것이다"라는 가정이 위험할 수 있음을 시사한다. 페르소나의 기술 수준과 상관없이 응답 시간 최적화는 모든 사용자 세그먼트에서 필수적인 요소임을 가설로 설정한다.
