# [문서 메타데이터]
- 출처: Staggers & Norcio (1993), "Mental models: concepts for human-computer interaction research", Int. J. Man-Machine Studies
- 핵심 참조 이론: Norman (1983a), "Some observations on mental models"
- 용도: 사용자 페르소나 생성 시 참고 — 사용자가 서비스를 어떻게 이해하고 있는지(멘탈 모델), 그 모델이 얼마나 성숙한지에 따른 행동 패턴 예측에 활용
- 도메인: HCI / 인지심리학 → 사용자가 서비스를 처음 접할 때와 익숙해진 후의 행동 차이 설명에 적용 가능

# CHUNK 1 — 핵심 개념: 멘탈 모델이란 무엇인가
**검색 키워드 힌트:** 멘탈 모델, 사용자 이해, 시스템 인식, 내부 표상

멘탈 모델(mental model)은 사용자가 시스템과 상호작용하면서 머릿속에 형성하는 내부 표상이다.
사용자는 이 모델을 바탕으로 "다음에 무슨 일이 일어날지"를 예측하고 행동한다.

Norman(1983a)의 핵심 구분:
- **개념 모델(conceptual model):** 디자이너/교육자가 설계한 "시스템이 실제로 어떻게 작동하는가"
- **시스템 이미지(system image):** 사용자에게 보이는 UI/인터페이스가 주는 인상
- **멘탈 모델(mental model):** 사용자가 실제로 머릿속에 갖고 있는 모델 (개념 모델과 다를 수 있음)

핵심: 사용자의 멘탈 모델은 디자이너의 의도와 다를 수 있고, 불완전하거나 틀릴 수 있다.
사용자는 자신이 이해한 대로 행동한다, 시스템이 실제로 작동하는 방식대로가 아니라.

→ **페르소나 적용:** "왜 저 기능을 저렇게 쓰지?"의 답은 대부분 사용자의 멘탈 모델에 있다.
페르소나를 만들 때 "이 사용자는 이 서비스를 어떻게 이해하고 있는가"를 명시해야 한다.

# CHUNK 2 — 멘탈 모델은 어떻게 형성되는가: 유추와 비유
**검색 키워드 힌트:** 첫 인상, 온보딩, 기존 경험 전이, 유추, 새 서비스 학습

사용자는 새로운 시스템을 접할 때 **기존에 알고 있는 것과의 유사성**으로 멘탈 모델을 만들기 시작한다.

예: 텍스트 에디터를 처음 쓰는 사용자는 자동으로 "타자기 모델"을 적용한다.
→ 타자기처럼 동작할 것이라는 가정 하에 행동하고, 다르게 동작하면 혼란스러워한다.

이 유추가 잘 맞으면 빠르게 학습이 이루어지고,
맞지 않으면 잘못된 가정이 굳어져 지속적인 오류의 원인이 된다.

중요한 점: 사용자는 유추가 틀렸다는 증거가 있어도 **기존 모델을 쉽게 버리지 않는다.**
오히려 새로운 정보를 자신의 기존 모델에 억지로 끼워 맞추려는 경향이 있다("creative comprehension").

→ **페르소나 적용:**
- 타겟 사용자가 기존에 유사 서비스를 사용한 경험이 있다면 → 그 서비스의 멘탈 모델이 전이된다
- 기존 모델과 새 서비스의 작동 방식이 다른 부분이 **정확히 막히는 지점**이 된다
- 예: 카카오페이 사용자가 토스를 처음 쓸 때, 송금 플로우가 다르면 혼란 발생

# CHUNK 3 — 미성숙한 멘탈 모델 vs 성숙한 멘탈 모델의 행동 차이
**검색 키워드 힌트:** 초보 사용자 행동, 전문가 사용자 행동, 오류 패턴, 문제 해결 전략

실제 SPSSX(통계 소프트웨어) 사용자 연구에서 관찰된 초보 vs 전문가의 차이:

**초보 사용자 (미성숙한 멘탈 모델):**
- 단계별 절차를 외워서 따라하는 "생존 전략" 사용
- 오류가 발생하면 완전히 멈춤 → 스스로 해결 불가
- 시스템 구조를 설명하지 못함 ("어떻게 돌아가는지 모르지만 일단 버튼 누름")
- 메뉴얼이 있어도 어느 섹션을 봐야 할지 모름
- 다른 사람의 도움에 크게 의존

**전문가 사용자 (성숙한 멘탈 모델):**
- 오류를 예상하고 시도-오류 방식으로 능동적으로 해결
- 시스템 구조를 추상적으로 이해하고 있어 새 기능도 유추 가능
- 메뉴얼에서 필요한 부분을 찾을 수 있음
- 오류 메시지를 해석할 수 있음

→ **페르소나 적용:**
- 초보 사용자 페르소나: "오류 메시지가 나오면 아무것도 못 하고 이탈"
- 중간 사용자 페르소나: "익숙한 기능은 빠르게 쓰지만 새 기능 앞에서는 초보로 돌아감"
- 전문가 사용자 페르소나: "시스템이 어떻게 작동하는지 알기 때문에 UI가 바뀌어도 금방 적응"

# CHUNK 4 — 멘탈 모델의 6가지 고유한 결함 (Norman, 1983)
**검색 키워드 힌트:** 사용자 오해, 잘못된 이해, UI 오류 원인, 사용자 실수 패턴

Norman(1983a)이 관찰한 멘탈 모델의 구조적 결함 — 이것이 UX 문제의 근본 원인:

1. **불완전함(Incomplete):** 시스템의 일부만 이해하고 있음
2. **실행 능력의 한계:** 머릿속에서 모델을 "돌려보는" 능력이 제한됨
3. **불안정함(Unstable):** 시간이 지나면 세부사항을 잊어버림
4. **경계 없음:** 유사한 기기나 서비스의 동작이 혼동됨
5. **비과학적 미신:** 실제로는 필요 없는 행동도 "혹시 몰라서" 계속 반복
6. **절약 지향(Parsimonious):** 머리 쓰는 것을 줄이기 위해 비효율적인 물리적 행동을 택함
   (예: 단축키 있어도 메뉴를 계속 클릭)

→ **페르소나 적용:**
- 결함 3(불안정): 가끔씩만 쓰는 사용자는 매번 처음부터 다시 배우는 것처럼 느낌
- 결함 5(미신): 사용자가 "꼭 이 순서로 해야 해"라고 믿는 불필요한 단계가 있을 수 있음
- 결함 6(절약): 사용자가 더 좋은 방법이 있어도 익숙한 방법을 고수하는 이유

# CHUNK 5 — 개념 모델을 먼저 보여주는 것이 왜 중요한가
**검색 키워드 힌트:** 온보딩 설계, 튜토리얼, 사용자 교육, 첫 사용 경험

연구들이 공통적으로 발견한 것: **절차적 설명보다 개념 모델을 먼저 제공한 사용자가 성과가 훨씬 좋다.**

이유: 개념 모델은 새로운 정보를 저장할 "지식의 틀"을 제공한다.
틀이 없으면 새로 배운 내용이 기억에 조직화되지 않고 흩어진다.

단, 조건이 있다:
- 개념 모델은 실제 사용 **전에** 제공되어야 효과가 있음 (사용 중이나 후에는 효과 감소)
- 직접 추론하게 두는 것보다 **명시적으로 보여주는 것**이 효과적
- 복잡한 과제일수록 개념 모델 여부가 성과 차이에 미치는 영향이 커짐

→ **페르소나 적용:**
- 복잡한 서비스(금융, B2B 툴 등)의 초보 사용자는 "일단 써보세요" 온보딩에서 멘탈 모델을 만들지 못함
- "이 서비스는 X를 Y하는 곳입니다" 형태의 한 줄 개념 설명이 이후 모든 상호작용의 질을 결정
- 페르소나가 서비스의 목적을 정확히 이해하고 있는지 여부가 UX 가설의 핵심 변수가 됨

# CHUNK 6 — "Cognitive Lock-up": 잘못된 멘탈 모델이 굳어지는 현상
**검색 키워드 힌트:** 잘못된 사용 습관, 오해 고착, 반복 오류, 사용자 재교육 어려움

Moray(1987)가 명명한 **"cognitive lock-up"**: 잘못된 멘탈 모델이 있는 사용자는
새로운 정보(오류 메시지, 안내 등)를 봐도 자신의 기존 모델을 고수하고 새 정보를 무시하거나 왜곡한다.

관련 현상들:
- **Creative comprehension:** 시스템이 예상과 다르게 동작해도 자신의 모델에 맞게 해석
- **Over-mapping:** 기존 서비스(예: 타자기)의 모든 규칙을 새 서비스에 억지로 적용
- **모델 간 간섭:** 여러 서비스를 쓰면서 명령어/UI 패턴이 섞임 (예: Ctrl+Z가 안 되는 서비스)

→ **페르소나 적용:**
- 경쟁 서비스의 헤비유저가 타겟이라면 → 그 서비스의 패턴이 강하게 전이되어 초기 마찰 발생
- 온보딩에서 "기존 서비스와 다른 점"을 명시적으로 알려주지 않으면 잘못된 멘탈 모델이 굳어짐
- 재방문 후에도 같은 오류를 반복하는 사용자 → cognitive lock-up 상태로 해석 가능
