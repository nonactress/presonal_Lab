# CHUNK 1 — 핵심 개념: 작업 기억(Working Memory)의 용량 한계
**검색 키워드 힌트:** 작업 기억, 단기 기억, 정보 처리 한계, 사용자 인지 한계

인간의 작업 기억(working memory)은 한 번에 보유할 수 있는 정보의 양이 엄격히 제한된다.
Miller(1956)의 고전 연구에서는 이 한계를 약 7개 내외(±2)로 제안했고,
이후 연구(Cowan, 2001)에서는 실제 한계가 더 엄격하여 약 3~4개의 의미 단위(chunk)에 가깝다는 것이 밝혀졌다.

핵심은 숫자 자체가 아니라, **한계가 존재한다**는 사실이다:
- 동시에 주의를 기울일 수 있는 항목의 수는 매우 적다
- 이 한계를 초과하면 항목들이 기억에서 탈락하거나 처리 자체가 실패한다

→ **페르소나 적용:** 사용자가 한 화면에서 동시에 고려해야 할 항목이 4개를 넘어가면
인지 부하가 급격히 증가하고, 결정 회피 또는 이탈로 이어질 수 있다.

# CHUNK 2 — 청킹(Chunking): 정보를 묶어서 한계를 극복하는 방법
**검색 키워드 힌트:** 청킹, 정보 묶음, 숙련 사용자, 패턴 인식, 메뉴 구조

Miller의 가장 중요한 기여는 숫자 7이 아니라 **청크(chunk) 개념**이다.
청크는 사용자에게 하나의 의미 단위로 인식되는 정보 묶음이다.

예시: "FBICIAUSA"는 9글자지만 FBI / CIA / USA로 파싱하면 3개의 청크가 된다.
→ 작업 기억의 한계는 글자 수나 항목 수가 아니라 **의미 단위의 수**에 적용된다.

청킹 능력은 경험과 지식에 따라 달라진다:
- **숙련 사용자:** 자주 쓰는 기능들을 하나의 청크로 묶어 인식 → 인지 부하 낮음
- **초보 사용자:** 각 항목을 개별적으로 처리 → 같은 화면도 훨씬 많은 인지 부하

→ **페르소나 적용:**
- 초보 사용자는 "설정 > 계정 > 보안 > 2단계 인증" 같은 경로도 4개의 독립 항목으로 처리한다.
- 숙련 사용자는 이 경로 전체를 하나의 청크로 기억하고 바로 이동한다.
- **같은 정보 구조가 두 페르소나에게 완전히 다른 난이도로 경험된다.**

# CHUNK 3 — 실제 작업 기억 한계는 3~4개: UX 설계 함의
**검색 키워드 힌트:** 선택지 수, 메뉴 항목, 정보 과부하, UI 설계 원칙

Miller의 7±2보다 더 정확한 한계는 **동시에 주의를 집중할 수 있는 항목 기준으로 약 3~4개**다.
(Cowan, 2001; Luck & Vogel, 1997)

전화번호가 전 세계적으로 2~4자리씩 묶어서 표기되는 것(예: 010-1234-5678)은
이 한계를 직관적으로 반영한 설계의 예다.

주의(attention)를 한꺼번에 모아야 하는 상황에서의 한계:
- 한 번에 비교해야 하는 선택지: 3~4개가 한계
- 동시에 기억해야 하는 단계: 3~4개가 한계
- 한 화면에서 처리해야 하는 결정: 3~4개가 한계

→ **페르소나 적용:**
- 초보 사용자 페르소나가 드롭다운에서 선택지가 10개 이상이면 → 결정 마비 또는 임의 선택 가능성
- 온보딩 단계가 7단계 이상이면 → 중간 이탈률 급증 예측 가능
- 한 폼에 입력 필드가 5개 이상이면 → 완료율 저하 가능성

# CHUNK 4 — 시간 한계 vs 항목 한계: 두 가지 다른 이탈 패턴
**검색 키워드 힌트:** 사용자 이탈 패턴, 시간 압박, 정보 망각, UX 실패 유형

작업 기억의 한계에는 두 가지 축이 있다:

**1. 항목 한계(item limit):** 동시에 유지할 수 있는 의미 단위의 수 제한 (약 3~4개)
**2. 시간 한계(time limit):** 주의를 기울이지 않으면 빠르게 소멸하는 정보

두 한계는 서로 다른 이탈 패턴을 만든다:
- **항목 한계 초과:** "선택지가 너무 많아서" / "뭘 골라야 할지 모르겠어서" 이탈
- **시간 한계 초과:** "하다가 잠깐 딴 데 보고 왔더니 어디까지 했는지 모르겠어서" 이탈

→ **페르소나 적용:**
- 복잡한 가입/설정 플로우에서 이탈한 사용자 → 두 원인 중 어느 쪽인지 구분해서 가설 수립
- 중간 저장 기능이 없는 멀티스텝 폼 → 시간 한계 이탈에 특히 취약한 구조

# CHUNK 5 — 청킹 능력은 학습된다: 반복 경험과 스키마의 관계
**검색 키워드 힌트:** 재방문 사용자, 학습 효과, 숙련도 향상, 사용 경험

청킹 능력은 고정된 것이 아니라 **경험을 통해 형성된다.**
같은 패턴을 반복적으로 접하면 여러 항목이 하나의 청크로 압축되고,
그 결과 작업 기억 부하가 줄어들어 더 복잡한 처리가 가능해진다.

이것은 Sweller의 스키마 개념과 직접 연결된다:
- 스키마 = 여러 항목이 하나의 청크로 묶인 구조
- 스키마가 형성될수록 인지 부하가 낮아지고 능숙한 사용이 가능해짐

→ **페르소나 적용:**
- **신규 사용자:** 청크 미형성 상태 → 모든 항목을 개별 처리 → 높은 인지 부하
- **재방문 사용자(3~5회):** 주요 플로우의 청크 형성 시작 → 핵심 경로는 빠르게, 새 기능은 여전히 어려움
- **장기 사용자:** 대부분의 기능이 청크화 → 레이아웃 변경이나 기능 이동에 오히려 더 강하게 반응(기존 청크 깨짐)
