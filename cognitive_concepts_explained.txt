1. COGNITIVE LOAD (인지 부하)
--------------------------------------------------------------------------------

[정의]
인간의 작업 기억(working memory)이 특정 순간에 처리해야 하는 정보의 양.
작업 기억의 용량은 고정되어 있고 한계가 있기 때문에, 이 한계를 초과하면
정보 처리가 실패하거나 사용자가 포기하게 된다.

[이론적 배경]
- Sweller(1988): 인지 부하가 높은 상태에서는 문제를 "풀어내는" 것 자체에
  용량이 소모되어 학습(패턴 인식, 스키마 형성)이 일어나지 않는다.
- Miller(1956) / Cowan(2001): 작업 기억은 한 번에 약 3~4개의 의미 단위
  (chunk)만 처리 가능하다.

[핵심 특성]
- 과제 자체의 객관적 어려움과 별개다.
  쉬운 과제도 동시에 많은 것을 요구하면 부하가 높아진다.
- 부하는 순간적이고 가변적이다.
  같은 사람도 상황에 따라 부하가 달라진다.
- 부하가 임계치를 넘으면 사용자는 멈추거나 이탈한다.

[UX에서 인지 부하가 높아지는 순간]
- 한 화면에서 동시에 고려해야 할 선택지가 4개 이상일 때
- 목표(내가 원하는 것)와 현재 상태(지금 어디 있는지)가 불명확할 때
- 행동의 결과를 예측할 수 없을 때 (버튼을 누르면 어떻게 될지 모를 때)
- 오류가 발생했는데 원인을 알 수 없을 때
- 다단계 플로우에서 이전 단계를 기억해야 할 때

[비유]
책상 위의 작업 공간. 공간이 고정되어 있어서,
올려놓을 것이 너무 많아지면 기존 것들이 바닥에 떨어진다.

2. MENTAL MODEL (멘탈 모델)
--------------------------------------------------------------------------------

[정의]
사용자가 시스템과 상호작용하면서 머릿속에 형성하는 내부 표상.
"이 서비스는 이렇게 작동할 것이다"라는 사용자 나름의 지도(map).
실제 시스템의 작동 방식과 일치하지 않을 수 있다.

[이론적 배경]
- Norman(1983): 디자이너의 개념 모델(conceptual model)과
  사용자의 멘탈 모델은 다를 수 있으며, 사용자는 자신의 멘탈 모델대로 행동한다.
- Staggers & Norcio(1993): 멘탈 모델은 기존 경험의 유추/비유를 통해 형성되며,
  불완전하고 불안정하고 틀릴 수 있다.

[핵심 특성]
① 불완전함: 시스템 전체가 아닌 일부만 이해하고 있다.
② 불안정함: 시간이 지나면 세부사항을 잊어버린다.
③ 비과학적 미신: 실제로 필요 없는 행동도 "혹시 몰라서" 반복한다.
④ 절약 지향: 머리 쓰는 것을 줄이기 위해 비효율적인 방식을 고수한다.
⑤ 전이(Transfer): 기존에 쓰던 서비스의 모델이 새 서비스에 그대로 적용된다.
⑥ 고착(Cognitive lock-up): 잘못된 모델이 있어도 새 정보를 무시하고 기존 모델을 유지한다.

[멘탈 모델 성숙도에 따른 행동 차이]

미성숙한 멘탈 모델 (초보 사용자):
- 단계별 절차를 외워서 따라하는 생존 전략 사용
- 오류 발생 시 완전히 멈춤, 스스로 해결 불가
- 시스템 구조를 설명하지 못함
- 도움(타인, 고객센터)에 크게 의존
- 새로운 상황에 유연하게 대응하지 못함

성숙한 멘탈 모델 (전문가 사용자):
- 시스템 구조를 추상적으로 이해하고 있어 새 기능도 유추 가능
- 오류를 예상하고 시도-오류 방식으로 능동적으로 해결
- 오류 메시지를 해석하고 원인을 추론할 수 있음
- 독립적으로 탐색하며 더 나은 방법을 찾으려 함

[UX 관점에서의 핵심]
사용자가 "왜 이상하게 쓰지?"의 답은 거의 항상 멘탈 모델에 있다.
사용자는 서비스가 실제로 작동하는 방식이 아니라
자신이 이해한 방식대로 행동한다.

[비유]
여행지 지도. 지도가 완벽하지 않아도 어느 정도 돌아다닐 수 있지만,
지도에 없는 골목에 들어서는 순간 완전히 길을 잃는다.

3. TECH LITERACY (기술 리터러시)
--------------------------------------------------------------------------------

[정의]
디지털 환경 전반에 대한 이해 수준과 능숙하게 탐색할 수 있는 능력.
단순히 "인터넷을 쓸 수 있는가"가 아니라,
새로운 서비스를 마주쳤을 때 스스로 이해하고 활용할 수 있는 역량.

[이론적 배경]
- Hargittai(2010): 인터넷 스킬은 젊은 층이라도 균일하지 않으며,
  사회경제적 배경(SES), 성별, 사용 자율성, 경험 연수에 의해 결정된다.
  "디지털 네이티브는 모두 능숙하다"는 가정은 틀렸다.

[핵심 특성]
- 개인의 노력이나 의지가 아닌 구조적 요인에 의해 결정된다.
- 한 번의 상호작용이 아닌 사람 자체의 배경 특성이다.
- 스킬 수준이 높을수록 더 다양한 방식으로 서비스를 활용한다.
- 자기 스킬에 대한 인식(자기효능감)이 실제 스킬과 다를 수 있다.
  (특히 여성 사용자는 실제 스킬이 있어도 자신을 낮게 평가하는 경향)

[리터러시 수준에 따른 행동 차이]

리터러시 낮음:
- 새로운 서비스를 매번 처음인 것처럼 경험
- 핵심 기능만 반복 사용, 새 기능 탐색 안 함
- 오류나 낯선 상황에서 빠르게 이탈
- 도움말, 공식 채널보다 주변 사람에게 의존
- "capital-enhancing" 활동(정보 검색, 금융 등)보다 익숙한 오락적 사용에 머묾

리터러시 높음:
- 처음 보는 서비스도 유사 서비스와의 비교로 빠르게 파악
- 다양한 기능을 능동적으로 탐색
- 오류 메시지를 단서로 활용
- 서비스를 더 다양한 목적으로 활용 (활용 다양성 높음)

[UX 관점에서의 핵심]
리터러시는 타겟 사용자를 정의할 때 가장 먼저 고려해야 할 변수다.
같은 UI를 보더라도 리터러시 수준에 따라 완전히 다른 경험을 한다.

[비유]
외국어 능력. 초급자는 간단한 문장도 하나하나 해독하지만,
고급자는 전체 맥락을 파악하며 자연스럽게 소통한다.

--------------------------------------------------------------------------------
세 개념의 인과 관계
--------------------------------------------------------------------------------

Tech Literacy → Mental Model → Cognitive Load

1단계: Tech Literacy가 Mental Model 형성의 질을 결정한다.
  리터러시가 높을수록 새 서비스를 접했을 때
  더 정확하고 풍부한 초기 멘탈 모델을 빠르게 형성할 수 있다.

2단계: Mental Model의 성숙도가 Cognitive Load를 결정한다.
  멘탈 모델이 잘 형성되어 있을수록 같은 UI를 보더라도
  인지 부하가 낮다. 모델이 있으면 "다음에 뭘 해야 하는지"를
  계산하는 데 작업 기억 용량을 쓰지 않아도 되기 때문이다.

3단계: Cognitive Load가 실제 행동(사용, 이탈, 오류)을 결정한다.
  부하가 임계치를 넘으면 사용자는 멈추거나 이탈한다.

요약:
"이 사용자의 리터러시 수준(Tech Literacy)이 낮기 때문에,
 이 서비스에 대한 멘탈 모델(Mental Model)이 빈약하게 형성되고,
 그 결과 이 화면에서 인지 부하(Cognitive Load)가 급증하여 이탈한다."


--------------------------------------------------------------------------------
페르소나 생성에서의 활용 방식
--------------------------------------------------------------------------------

Tech Literacy  → "이 사람은 어떤 사람인가"를 정의할 때 사용
                  (배경 특성, 스킬 수준, 사용 경험)

Mental Model   → "이 사람은 이 서비스를 어떻게 이해하는가"를 정의할 때 사용
                  (기존 경험 전이, 오해 지점, 멘탈 모델 성숙도)

Cognitive Load → "이 사람은 어디서 구체적으로 막히는가"의 가설을 생성할 때 사용
                  (특정 화면, 특정 단계, 특정 순간)
