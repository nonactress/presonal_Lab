# Frontend 기술 스택 (Pivot: Dev-First UI)

## 프레임워크: Gradio

### 선택 이유
- **코드 입력 최적화:** `gr.Code()` 컴포넌트를 통해 문법 강조(Syntax Highlighting)가 포함된 코드 입력창 제공 가능
- **빠른 프로토타이핑:** 2주 MVP 기한 내에 "입력 -> 분석 -> 복사" 흐름을 가장 빠르게 구현

---

## 구현할 UI 흐름 (MVP)

### 1단계 — 코드 입력 및 설정
```
[코드 입력창 (React/Tailwind)]
[타겟 페르소나 선택 (Dropdown)]
  - 60대 어르신 (저리터러시)
  - 20대 대학생 (일반)
  - 직접 입력...
[분석 시작 버튼]
```

### 2단계 — UX 분석 리포트 및 Fix
```
[혼란 지수 대시보드]
  - 전체 UX 점수
  - 주요 위반 항목 (예: "버튼 크기 미달", "대비 부족")

[Vibe Coding Fix Zone]
  - 각 문제점별로 대응하는 'Cursor/v0용 수정 프롬프트'
  - [Prompt Copy] 버튼 (클릭 시 바로 복사)

[Mental Simulation]
  - 사용자가 이 코드를 보며 느낄 생각을 텍스트로 표시
```
