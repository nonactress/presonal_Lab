# M1 — Code & UI Analyzer (Pivot)

## 목적

입력받은 프론트엔드 소스코드(React, Tailwind 등)를 분석하여 UI의 구조적/시각적 의미를 파악한다.
브라우저 렌더링 없이 LLM의 'Mental Rendering'을 통해 사용자가 보게 될 화면을 추론한다.

---

## 입출력

|            | 내용                                       |
| ---------- | ---------------------------------------- |
| **Input**  | 프론트엔드 소스코드 (TSX, JSX, HTML 등), 타겟 태스크 |
| **Output** | UI 요소 맵 (버튼, 입력창의 역할 및 스타일링 정보), 시각적 위계 분석 결과 |

---

## 구현 상세

### Step 1 — 코드 시맨틱 분석

LLM에게 코드를 전달하고, 사용자가 시각적으로 인지하게 될 구조를 추출한다.

```python
def analyze_code_structure(source_code):
    prompt = f"""
    아래 프론트엔드 코드를 분석해서 사용자가 보게 될 UI 구조를 설명해줘.
    
    분석 항목:
    1. 주요 컴포넌트의 역할 (예: 로그인 폼, 내비게이션 바)
    2. 시각적 강조 요소 (Tailwind 클래스 분석: 색상, 크기, 애니메이션 등)
    3. 인터랙션 요소 (클릭 가능 여부, 입력 필드 등)
    4. 코드 라인 번호 매핑 (나중에 수정을 위해 필요)
    
    Source: {source_code}
    """
    return llm.call(prompt)
```

### Step 2 — 시뮬레이션용 데이터 구조화

M3 시뮬레이션 엔진이 이해할 수 있는 형태의 JSON으로 변환한다.

```json
{
  "components": [
    {
      "type": "button",
      "label": "회원가입 완료",
      "styling": {"color": "blue-500", "size": "small"},
      "line_number": 45,
      "context": "폼 하단의 최종 제출 버튼"
    }
  ],
  "visual_hierarchy": "상단 로고보다 하단 광고 배너가 더 눈에 띔",
  "potential_issues": ["텍스트 대비 낮음 (text-gray-300)"]
}
```

---

## 장점 (Vibe Coder 타겟)

- **정확한 수정 포인트:** "저 버튼"이 아니라 "45번 라인의 버튼"이라고 지칭 가능.
- **실시간성:** 브라우저 로딩 대기 없이 코드 복사 즉시 분석 가능.
- **의도 파악:** 코드 내 주석이나 변수명을 통해 개발자의 UX 의도를 더 깊이 이해 가능.

---

## 우리가 직접 구현하는 것

- 코드 라인 보존 파싱 전략
- 소스코드 기반 시각적 추론(Mental Rendering) 프롬프트 템플릿
- Tailwind/CSS 스타일링 가중치 분석 로직
