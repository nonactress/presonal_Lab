# C4 — 한국 문화 맥락

## 문제

서양 UX 연구를 그대로 쓰면 한국 사용자 행동과 다르다.
한국 데이터가 없는 feature를 억지로 채우면 틀린 시뮬레이션이 나온다.

---

## 서양 연구 vs 한국 실제 차이

| 행동       | 서양 연구 결론     | 한국 실제           | 출처 필요    |
| -------- | ------------ | --------------- | -------- |
| 정보 밀도 선호 | 심플한 화면 선호    | 정보 많은 화면을 신뢰함   | ⚠️ 검증 필요 |
| 로딩 인내심   | 3초 이탈 기준     | 더 짧음 (빨리빨리)     | ⚠️ 검증 필요 |
| 에러 발생 시  | 에러 메시지 읽고 대처 | 고객센터 전화 (외부 의존) | ⚠️ 검증 필요 |
| 개인정보 동의  | 꼼꼼히 읽는 비율 높음 | 전체 동의 누르는 비율 높음 | ⚠️ 검증 필요 |
| 앱 아이콘 배치 | 기능별 그룹화 선호   | 자주 쓰는 것 전면 배치   | ⚠️ 검증 필요 |

→ **"검증 필요" 항목은 우리가 직접 데이터를 수집하거나 논문에서 확인해야 함**

---

## 한국 특화 Feature 후보

```python
"korean_context": {

    # 빨리빨리 — 대기/로딩에 대한 인내심
    "bballi_bballi": float,       # 0=천천히, 1=즉각 반응 요구

    # 정보 밀도 선호
    "info_density_pref": float,   # 0=심플, 1=많은 정보 한번에

    # 외부 의존 성향 — 막히면 전화/검색 vs 혼자 해결
    "external_help_tendency": float,

    # 공식 인증 신뢰 — 뱅킹앱 로고, 공공기관 UI 패턴
    "authority_ui_trust": float,

    # 카카오/네이버 UI 패턴 친숙도 (한국 표준 멘탈 모델)
    "kakao_naver_familiarity": float,
}
```

---

## 데이터 없는 Feature 처리 원칙

한국 특화 데이터가 없으면:

```
1순위: 한국 연구 논문에서 찾기
2순위: 국내 앱스토어 리뷰에서 패턴 추출
3순위: 서양 연구에 보정 계수 적용 (보정 근거 명시)
4순위: unknown 처리 — 시뮬레이션에서 "보통" 기본값 + 경고 표시

절대 금지: 근거 없이 값 생성
```
